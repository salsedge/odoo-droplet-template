  <files>scripts/06-backup-daily.sh, scripts/07-sync-offsite.sh</files>
  <action>
Create `scripts/06-backup-daily.sh` — the daily backup script called by cron. Must include:

**Header and setup:**
- Bash strict mode (set -euo pipefail)
- Requirement ID comments (BACK-01, BACK-03)
- Constants: BACKUP_DIR=/mnt/odoo-prod-data/backups, ENV_FILE=/opt/odoo/.env, STATUS_FILE=/opt/odoo/backup-status.json, LOG_TAG=odoo-backup
- Track start time for duration reporting
- Trap for error handling that writes failure status + sends email on any error

**Pre-flight checks:**
- Verify running as root
- Check Docker is running and odoo-db container is healthy (`docker inspect --format '{{.State.Health.Status}}' odoo-db`)
- Check available disk space on /mnt/odoo-prod-data (fail if less than 2GB free)
- Create backup directories if they don't exist: ${BACKUP_DIR}/{daily,weekly}

**Credential extraction (safe for special chars):**
- Extract POSTGRES_USER, POSTGRES_DB from .env using `grep '^KEY=' | cut -d'=' -f2-` pattern (NOT source .env — established in 03-deploy-stack.sh)

**Database backup (BACK-01):**
- Use `docker exec odoo-db pg_dump -U "${POSTGRES_USER}" -d "${POSTGRES_DB}" -Fc --no-owner --no-privileges` piped to temp file
- Write to temp file first, rename on success (prevents corrupt partial backups)
- Naming: odoo-db-YYYY-MM-DD.dump

**Filestore backup:**
- `tar -czf` of /mnt/odoo-prod-data/odoo-filestore/ excluding sessions and addons subdirectories
- Write to temp file first, rename on success
- Naming: odoo-files-YYYY-MM-DD.tar.gz

**Weekly promotion (BACK-03):**
- If day of week is Sunday (date +%u == 7), hard link (`cp -l`, falling back to `cp`) daily files into weekly/ directory

**Retention cleanup (BACK-03):**
- Run BEFORE backup creation to free space first
- Daily: `find ${BACKUP_DIR}/daily/ -name "odoo-*" -mtime +7 -delete`
- Weekly: `find ${BACKUP_DIR}/weekly/ -name "odoo-*" -mtime +28 -delete`

**Status file (Phase 5 prep):**
- Write JSON to /opt/odoo/backup-status.json with: status (0=OK, 2=CRITICAL per Nagios convention), message, timestamp (ISO 8601), db_size_bytes, files_size_bytes, duration_seconds
- chmod 644 on status file (readable by Icinga2 agent)

**Email notification on failure:**
- On error trap: send email via `mail` command (provided by msmtp-mta) to ALERT_EMAIL (from .env)
- Include error message, timestamp, hostname

Create `scripts/07-sync-offsite.sh` — rclone offsite sync script called by separate cron. Must include:

**Header and setup:**
- Bash strict mode, requirement ID comment (BACK-02)
- Constants: RCLONE_CONF=/opt/odoo/rclone.conf, BACKUP_DIR=/mnt/odoo-prod-data/backups/daily

**Sync logic:**
- Use `rclone copy` (NOT sync — sync would delete older remote backups)
- Organize by year/month: spaces:odoo-prod-backups/YYYY/MM/
- Include only today's files: `--include "odoo-*-$(date +%Y-%m-%d).*"`
- Log output for cron capture

**Error handling:**
- Check rclone exit code
- Append sync status to the backup status file or write a separate sync-status.json
- Send email on failure via mail command

Both scripts must be executable (chmod +x pattern in header comments).

<task type="auto">
  <name>Task 2: Create restore and verification script</name>
  <files>scripts/08-restore-backup.sh</files>
  <action>
Create `scripts/08-restore-backup.sh` — restore + verification script. Must include:

**Header and setup:**
- Bash strict mode, requirement ID comment (BACK-04)
- Usage function showing all flags: --file PATH, --from-spaces, --date YYYY-MM-DD, --verify-only, --production, --help
- Argument parsing via while/case loop
- Constants: BACKUP_DIR=/mnt/odoo-prod-data/backups, RCLONE_CONF=/opt/odoo/rclone.conf, ENV_FILE=/opt/odoo/.env

**Source selection logic:**
1. If `--file PATH` provided: use that specific dump file
2. If `--from-spaces --date YYYY-MM-DD`: fetch from DO Spaces via `rclone copy spaces:odoo-prod-backups/YYYY/MM/odoo-db-DATE.dump` to a temp directory, also fetch matching filestore archive
3. If neither: find most recent .dump file in ${BACKUP_DIR}/daily/

**Verify-only mode (default if no --production flag):**
- Start temp PostgreSQL 18 container: `docker run -d --name odoo-restore-test -e POSTGRES_USER=restore_test -e POSTGRES_PASSWORD=test_only -e POSTGRES_DB=odoo_restore postgres:18`
- Wait for container healthy (poll pg_isready with timeout)
- Restore: `docker exec -i odoo-restore-test pg_restore -U restore_test -d odoo_restore --no-owner --clean --if-exists < DUMP_FILE`
- Verification queries (all via docker exec psql):
  1. Count tables in public schema (expect > 100 for Odoo)
  2. Check key Odoo tables exist: res_users, res_partner, crm_lead, project_project, ir_module_module
  3. Row counts for res_users and res_partner (expect > 0)
  4. Check ir_module_module has installed CRM and Project modules
- Print verification summary with PASS/FAIL per check
- Start temp Odoo container against restored DB to verify boot (optional, nice-to-have — if Odoo image available, boot with --stop-after-init against the temp PG)
- Cleanup: docker stop + rm temp containers
- Exit 0 if all checks pass, exit 1 if any fail

**Production mode (--production flag):**
- Confirm prompt: "This will stop Odoo, drop the database, and restore from backup. Continue? [y/N]"
- Stop Odoo container: `docker compose -f /opt/odoo/docker-compose.yml stop odoo`
- Read production PG credentials from .env
- Restore into production PG container using docker exec pg_restore
- If filestore archive exists alongside DB dump: extract to /mnt/odoo-prod-data/odoo-filestore/
- Restart Odoo: `docker compose -f /opt/odoo/docker-compose.yml start odoo`
- Wait for health check, print result

Script must handle errors gracefully — always clean up temp containers even on failure (trap).
