# Plan 02-01: Host Hardening & Docker Installation

**Phase:** 02 — Hardened Application Stack
**Wave:** 1 (prerequisite for 02-02 and 02-03)
**Requirements:** HARD-01, HARD-02, HARD-03, HARD-04, HARD-05, HARD-06, HARD-07, DOCK-01, DOCK-02, DOCK-07

## Goal

Harden the bare Ubuntu 24.04 droplet to PCI-DSS baseline and install Docker CE with production configuration. After this plan completes, the host has: non-standard SSH (port 9292, key-only), UFW firewall, fail2ban, kernel hardening, auditd, unattended upgrades, and Docker CE with `iptables: false` and log rotation.

## Tasks

### Task 1: Update Terraform firewall for SSH port 9292

The DO cloud firewall operates above the droplet — it must allow port 9292 before SSH can be changed on the host.

**Files modified:**
- `infra/variables.tf` — Add `ssh_port` variable (default: 9292)
- `infra/main.tf` — Update firewall SSH rule `port_range` to use `var.ssh_port`
- `infra/terraform.tfvars.example` — Add `ssh_port = 9292`

**Verification:** `terraform plan` shows firewall rule change from port 22 to 9292.

### Task 2: Create host hardening script + config files

**Files created:**
- `scripts/01-harden-host.sh` — Main hardening script (runs as root)
- `config/sshd-hardening.conf` — SSH drop-in: port 9292, key-only, no root, idle timeout, max 3 auth tries
- `config/sysctl-hardening.conf` — Kernel params: SYN cookies, no IP forwarding, no ICMP redirects, martian logging, reverse path filter
- `config/jail.local` — fail2ban: SSH jail (port 9292, 5 retries/10min ban) + Odoo login jail (pre-configured, activates when logs exist)
- `config/audit.rules` — auditd: PCI-DSS 10.2.1–10.2.7, Docker/SSH/firewall change tracking, immutable config

**Script behavior:**
1. System update + install packages (ufw, fail2ban, auditd, unattended-upgrades)
2. Deploy SSH hardening config, create `deploy` user with sudo + SSH keys, restart sshd
3. Configure UFW (default-deny, allow 9292/80/443)
4. Deploy fail2ban with SSH + Odoo login jails
5. Apply sysctl kernel hardening
6. Configure unattended security upgrades
7. Restrict file permissions on sensitive system files
8. Deploy auditd rules and load

### Task 3: Create Docker CE installation script + daemon.json

**Files created:**
- `scripts/02-install-docker.sh` — Install Docker CE + Compose v2 from official Docker apt repo
- `config/daemon.json` — `iptables: false`, json-file log driver with 10MB/3-file rotation

**Script behavior:**
1. Remove conflicting packages (docker.io, podman, etc.)
2. Add Docker's official GPG key and apt repository
3. Install docker-ce, docker-ce-cli, containerd.io, docker-compose-plugin
4. Deploy daemon.json and restart Docker
5. Add deploy user to docker group
6. Verify with hello-world container

## Execution Order

Run `terraform apply` first (Task 1), then SCP files to droplet and run scripts sequentially:
```
terraform -chdir=infra apply
scp -r config/ scripts/ root@<droplet-ip>:/tmp/odoo-setup/
ssh root@<droplet-ip> 'bash /tmp/odoo-setup/scripts/01-harden-host.sh'
ssh -p 9292 deploy@<droplet-ip> 'sudo bash /tmp/odoo-setup/scripts/02-install-docker.sh'
```
