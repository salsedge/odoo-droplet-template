# Plan 02-03: Nginx Reverse Proxy & SSL

**Phase:** 02 — Hardened Application Stack
**Wave:** 2 (parallel with 02-02)
**Depends on:** 02-01
**Requirements:** PROXY-01, PROXY-02, PROXY-03, PROXY-04, PROXY-05

## Goal

Install Nginx on the host as a reverse proxy to Odoo, obtain a Let's Encrypt SSL certificate via HTTP-01 challenge, and configure HTTPS with security headers. After this plan, `https://[domain]` loads Odoo with valid SSL, HSTS, HTTP→HTTPS redirect, and database manager routes blocked.

## Tasks

### Task 1: Create Nginx configuration files

**Files created:**
- `config/nginx/odoo-pre-ssl.conf` — Temporary port-80 config serving only `/.well-known/acme-challenge/` for certbot
- `config/nginx/odoo.conf` — Full SSL config with:
  - Upstream definitions for Odoo (8069) and longpolling (8072)
  - HTTP→HTTPS redirect (port 80 → 443)
  - SSL with TLS 1.2/1.3, modern cipher suite, OCSP stapling
  - HSTS: `max-age=31536000` (no includeSubDomains)
  - Security headers: X-Frame-Options SAMEORIGIN, X-Content-Type-Options nosniff, Referrer-Policy strict-origin-when-cross-origin, CSP for Odoo compatibility
  - `/web/database` → 403 (PROXY-04)
  - `/websocket` → odoo-chat upstream (longpolling)
  - Proxy buffers sized for Odoo (16×64k, 128k buffer)
  - 100MB client_max_body_size for file uploads
  - Gzip compression for text/CSS/JS/JSON

**Design notes:**
- DOMAIN_PLACEHOLDER in config files is replaced by `sed` during deployment
- CSP allows `unsafe-inline` and `unsafe-eval` for Odoo's JavaScript framework
- SAMEORIGIN frame policy allows Odoo's internal iframe usage

### Task 2: Create Nginx installation and certbot setup script

**Files created:**
- `scripts/04-setup-nginx.sh` — Takes domain as argument, optional email

**Script behavior:**
1. Install nginx and certbot via apt
2. Deploy pre-SSL config (sed domain substitution), start Nginx on port 80
3. Run `certbot certonly --webroot` for HTTP-01 challenge
4. Deploy full SSL config (sed domain substitution), reload Nginx
5. Create systemd timer for certbot auto-renewal (twice daily with 1h random delay)
6. Enable and start the timer

**Certbot auto-renewal (PROXY-05):**
- `certbot-renewal.service` — oneshot, runs `certbot renew --quiet --deploy-hook "systemctl reload nginx"`
- `certbot-renewal.timer` — runs at 00:00 and 12:00 with RandomizedDelaySec=3600

## user_setup

2. Wait for DNS propagation (verify with `dig <domain>`)
