# Plan 02-02: Docker Application Stack

**Phase:** 02 — Hardened Application Stack
**Wave:** 2 (parallel with 02-03)
**Depends on:** 02-01
**Requirements:** DOCK-03, DOCK-04, DOCK-05, DOCK-06, ODOO-01, ODOO-02, ODOO-03, ODOO-04, ODOO-05, PG-01, PG-02, PG-03, PG-04

## Goal

Deploy containerized Odoo 19 + PostgreSQL 16 on the hardened host. Both services are healthy, data is persisted on Block Storage Volume, PostgreSQL is isolated to the backend Docker network, and Odoo listens on localhost:8069 only (ready for Nginx in 02-03).

## Tasks

### Task 1: Create Docker Compose configuration

**Files created:**
- `config/docker-compose.yml` — Two services (odoo, db), two networks (frontend, backend)
- `config/.env.example` — Template for POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB, ODOO_ADMIN_PASSWORD

**Design:**
- PostgreSQL: `postgres:16` image, backend network only, no published ports, 1.2GB/0.5CPU limit
- Odoo: `odoo:19` image, frontend+backend networks, `127.0.0.1:8069` and `127.0.0.1:8072` published
- Health checks: `pg_isready` for PostgreSQL, `curl localhost:8069/web/health` for Odoo
- Backend network is `internal: true` (no outbound internet)
- Odoo depends on db with `condition: service_healthy`

### Task 2: Create Odoo and PostgreSQL configuration files

**Files created:**
- `config/odoo.conf` — 3 workers + 1 cron, 768MB soft / 1024MB hard memory limits, proxy_mode=True, list_db=False, filestore at /var/lib/odoo
- `config/postgresql.conf` — 256MB shared_buffers, 8MB work_mem, 50 max_connections, slow query logging (>1s)

**Tuning rationale (s-2vcpu-4gb droplet):**
- PostgreSQL: 256MB shared_buffers ≈ 25% of 1.2GB container limit
- Odoo: 3 workers = (2 × CPU) + 1, fits within 2GB container limit (3 × 768MB soft ≤ 2048MB)
- 50 max_connections leaves headroom beyond Odoo's db_maxconn=32

### Task 3: Create stack deployment script

**Files created:**
- `scripts/03-deploy-stack.sh` — Directory setup, config deployment, docker compose up, module init

**Script behavior:**
1. Verify .env exists (not .env.example)
2. Create directories on Block Storage Volume: `/mnt/odoo-prod-data/{postgres-data,odoo-filestore}`
3. Set ownership: postgres-data → uid 999, odoo-filestore → uid 101
4. Copy configuration to /opt/odoo, set .env to mode 600
5. Pull images and start stack
6. Wait for health checks (120s timeout)
7. Initialize Odoo with `-i crm,project --stop-after-init`
8. Restart Odoo to enter multi-worker mode

## user_setup

Before running the deploy script:
```
cp config/.env.example config/.env
# Edit config/.env with strong, unique passwords
chmod 600 config/.env
```
