  <action>
    **scripts/infra-audit.sh**: Bash script that verifies server hardening over SSH. Uses the same script conventions as the project's `scripts/` directory (numbered checks, PASS/FAIL output, summary at end).

    Script structure:
    - Shebang: `#!/usr/bin/env bash`
    - Usage: `bash scripts/infra-audit.sh [--host HOST] [--port PORT] [--user USER]`
    - Defaults: reads INFRA_SSH_HOST, INFRA_SSH_PORT (default 9292), INFRA_SSH_USER (default deploy) from env or CLI args
    - If no host provided, print usage and exit 1

    Checks (each outputs `[PASS]` or `[FAIL]` with description):
    1. **HARD-01: SSH on non-standard port** — verify SSH connects on configured port (not 22)
    2. **HARD-01: Root login disabled** — `ssh ... "sudo grep -E '^PermitRootLogin' /etc/ssh/sshd_config.d/99-hardening.conf"` should show `no`
    3. **HARD-01: Password auth disabled** — grep sshd config for `PasswordAuthentication no`
    4. **HARD-02: UFW active** — `sudo ufw status` should show "active"
    5. **HARD-02: UFW rules correct** — check for 9292/tcp, 80/tcp, 443/tcp allow rules
    6. **HARD-03: fail2ban running** — `systemctl is-active fail2ban` should be "active"
    7. **HARD-03: fail2ban SSH jail active** — `sudo fail2ban-client status sshd` should show jail
    8. **HARD-05: Unattended upgrades enabled** — check `apt-config dump` or service status
    9. **HARD-07: auditd running** — `systemctl is-active auditd` should be "active"
    10. **DOCK-02: Docker iptables disabled** — `cat /etc/docker/daemon.json` should contain `"iptables": false`
    11. **DOCK-06: Docker containers healthy** — `docker ps --format '{{.Names}} {{.Status}}'` should show healthy Odoo and PG containers

    Summary at end: `N/M checks passed`. Exit 0 if all pass, exit 1 if any fail.

    Use SSH with options: `-o ConnectTimeout=10 -o StrictHostKeyChecking=accept-new -p $PORT $USER@$HOST`.

    Each check runs via a helper function `run_check "DESCRIPTION" "COMMAND" "EXPECTED_PATTERN"` that SSHes in, runs the command, greps for the pattern, and prints PASS/FAIL.

    **package.json updates**: Add/update npm scripts:
    - `setup:local` — `docker compose up -d --wait && npx playwright test --project=local tests/setup/ && node -e "console.log('\\n--- UAT READY ---\\nOdoo: http://localhost:8069\\nAdmin: ' + process.env.ADMIN_LOGIN + '\\nModules: CRM, Project installed\\nTest user: ' + (process.env.TEST_USER_LOGIN || 'not configured') + '\\n\\nTo tear down: npm run teardown:local')"` — starts stack, runs setup, prints UAT handoff summary
    - `teardown:local` — `docker compose down -v` (already exists, verify)
    - `audit:infra` — `bash scripts/infra-audit.sh`
    - `audit:ui` — `npx playwright test --project=production tests/audit/odoo-ui-audit.spec.ts`
    - `audit:headers` — `npx playwright test --project=production tests/audit/http-headers.spec.ts`
    - `audit:all` — `npm run audit:ui && npm run audit:headers && npm run audit:infra`

    Make infra-audit.sh executable: `chmod +x scripts/infra-audit.sh`.
  </action>
