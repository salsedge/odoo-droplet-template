<task type="auto">
  <name>Task 1: Start local stack, run test suite, collect results</name>
  <files></files>
  <action>
    1. Copy `odookit/.env.example` to `odookit/.env` (if not already present), filling in:
       - ADMIN_LOGIN=admin
       - ADMIN_PASSWORD=admin (Odoo default for local dev)
       - LOCAL_ODOO_URL=http://localhost:8069
       - LOCAL_POSTGRES_USER=odoo
       - LOCAL_POSTGRES_PASSWORD=odoo_local_dev
       - LOCAL_POSTGRES_DB=odoo
       - TEST_USER_LOGIN=testuser@example.com
       - TEST_USER_PASSWORD=TestPass123!

    2. Start the local stack: `cd odookit && docker compose up -d --wait`
    3. Wait for Odoo: poll `http://localhost:8069/web/health` until ready
    4. Run setup tests first: `npx playwright test --project=local tests/setup/`
    5. Run smoke tests: `npx playwright test --project=local tests/smoke/`
    6. Run workflow tests: `npx playwright test --project=local tests/workflows/`
    7. Run audit tests (UI only, skip header SSL checks): `npx playwright test --project=local tests/audit/`
    8. Generate HTML report: `REPORT=1 npx playwright test --project=local`
    9. Capture any failures for checkpoint review

    If any tests fail, investigate the failure, fix page objects or test logic as needed, and re-run. Record which fixes were required.
  </action>
  <verify>
    - Docker containers are running: `docker ps` shows odookit-odoo and odookit-db healthy
    - All test suites pass or have documented skip reasons
    - HTML report generated in `odookit/reports/`
  </verify>
  <done>
    Test suite runs to completion against the local stack. Failures are either fixed or documented with reasons.
  </done>
