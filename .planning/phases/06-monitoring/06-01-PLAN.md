**Script structure:**
1. Shebang `#!/usr/bin/env bash`, `set -uo pipefail`
2. Constants: `STATE_OK=0`, `STATE_WARNING=1`, `STATE_CRITICAL=2`, `STATE_UNKNOWN=3`
3. Default thresholds: `CONTAINERS="odoo-app,odoo-db"`, `WARN_RESTARTS=2`, `CRIT_RESTARTS=5`

**Check logic per container (worst status wins across all containers):**
- `docker inspect --format='{{.State.Running}}' "$container"` â€” if command fails (container not found) or returns != "true", exit CRITICAL immediately
- `docker inspect --format='{{if .State.Health}}{{.State.Health.Status}}{{else}}no-healthcheck{{end}}' "$container"` â€” if "unhealthy", exit CRITICAL immediately
- `docker inspect --format='{{.RestartCount}}' "$container"` â€” >= CRIT_RESTARTS sets status CRITICAL, >= WARN_RESTARTS sets status WARNING

**Output format (Nagios Plugin API):**
- OK: `OK - All containers healthy (odoo-app: healthy/0, odoo-db: healthy/0) | 'odoo-app_restarts'=0;2;5;0; 'odoo-db_restarts'=0;2;5;0;`
- WARNING: `WARNING - odoo-app restart count 3 | 'odoo-app_restarts'=3;2;5;0; 'odoo-db_restarts'=0;2;5;0;`
- CRITICAL: `CRITICAL - Container odoo-app is not running`

**Error handling:**
- If `docker` command not found, exit UNKNOWN with message "docker command not found"
- If docker inspect fails (permission denied), exit UNKNOWN with message suggesting `nagios` user needs docker group membership
- Collect status details for ALL containers before deciding final exit code (don't short-circuit on first warning â€” but DO short-circuit on immediate criticals like container not running or unhealthy)

Create `monitoring/plugins/check_postgres_health` â€” a bash Nagios/Icinga2 check plugin that monitors PostgreSQL metrics via `docker exec` into the odoo-db container.

**Script structure:**
1. Shebang `#!/usr/bin/env bash`, `set -uo pipefail`
2. Constants: `STATE_OK=0`, `STATE_WARNING=1`, `STATE_CRITICAL=2`, `STATE_UNKNOWN=3`
3. Default thresholds:
   - `CONTAINER="odoo-db"`
   - `PG_USER="${PG_USER:-odoo}"`, `PG_DB="${PG_DB:-odoo}"`
   - Connections: `WARN_CONN=35`, `CRIT_CONN=45` (of max_connections=50)
   - DB size: `WARN_SIZE=5368709120` (5GB), `CRIT_SIZE=10737418240` (10GB)
   - Latency: `WARN_LATENCY=100` (ms), `CRIT_LATENCY=500` (ms)
   - Cache hit: `WARN_CACHE=90` (%), `CRIT_CACHE=80` (%)
4. `usage()` function with all flags: `--container`, `--pg-user`, `--pg-db`, `--warn-conn`, `--crit-conn`, `--warn-size`, `--crit-size`, `--warn-latency`, `--crit-latency`, `--warn-cache`, `--crit-cache`, `--help`
5. Argument parsing via `while` loop with `case`

**Helper function:**
```
pg_query() {
  docker exec -i "$CONTAINER" psql -U "$PG_USER" -d "$PG_DB" -t -A -c "$1" 2>/dev/null
}
```

**Metrics collected (all in one invocation):**

1. **Active connections:** `SELECT count(*) FROM pg_stat_activity;`
   - Compare against WARN_CONN / CRIT_CONN

2. **Database size (bytes):** `SELECT pg_database_size(current_database());`
   - Compare against WARN_SIZE / CRIT_SIZE
   - Convert to human-readable in status text (MB/GB)

3. **Query latency (ms):** Measure wall-clock time of `pg_query "SELECT 1;"`
   - Use `date +%s%N` before and after, calculate `(end - start) / 1000000`
   - Compare against WARN_LATENCY / CRIT_LATENCY
   - Note: includes docker exec overhead (~20-50ms), thresholds account for this

4. **Cache hit ratio (%):** `SELECT ROUND(100.0 * sum(blks_hit) / (sum(blks_hit) + sum(blks_read) + 1), 2) FROM pg_stat_database WHERE datname = current_database();`
   - The `+1` in denominator prevents division by zero on fresh databases
   - Compare: below WARN_CACHE sets WARNING, below CRIT_CACHE sets CRITICAL
   - Note: cache hit is inverted â€” LOWER is worse (unlike other metrics where higher is worse)

**Worst-status-wins logic:**
Track overall_status, update to max(current, new_status) after each metric evaluation. Don't short-circuit â€” collect all metrics for complete perfdata output.

**Output format (Nagios Plugin API):**
- OK: `OK - PG healthy (conn:12, size:256MB, latency:35ms, cache:99.50%) | 'connections'=12;35;45;0;50 'db_size'=268435456B;5368709120;10737418240;0; 'query_latency'=35ms;100;500;0; 'cache_hit_ratio'=99.50%;90;80;0;100`
- WARNING: `WARNING - Connections 37/50 | 'connections'=37;35;45;0;50 ...`

**Error handling:**
- If `docker exec` fails (container not running), exit UNKNOWN with clear message
- If individual query fails, report that metric as UNKNOWN but continue collecting other metrics
- If docker command not found, exit UNKNOWN
- Detect permission denied and suggest docker group membership for nagios user

Make the file executable. Include header comment block with: purpose, requirement ID (MON-03), Nagios Plugin API reference, usage example, and note about docker exec latency overhead.
