  <name>Task 1: Create Terraform project scaffold with providers, backend, and .gitignore</name>
  <files>infra/providers.tf, infra/backend.tf, infra/.gitignore</files>
  <action>
Create the `infra/` directory and three files:

**infra/providers.tf:**
- `terraform` block with `required_version = ">= 1.6.3"`
- `required_providers` block: `digitalocean` source `"digitalocean/digitalocean"` version `"~> 2.0"`
- `provider "digitalocean"` block with `token = var.do_token`

**infra/backend.tf:**
- `terraform` block with `backend "s3"` configuration per RESEARCH.md Pattern 3
- `endpoints = { s3 = "https://nyc3.digitaloceanspaces.com" }`
- `bucket = "odoo-prod-tfstate"` (operator will change this during bootstrap)
- `key = "terraform.tfstate"`
- All five `skip_*` flags set to `true`: `skip_credentials_validation`, `skip_requesting_account_id`, `skip_metadata_api_check`, `skip_region_validation`, `skip_s3_checksum`
- `region = "us-east-1"` (required by S3 backend but unused by DO)
- Add a comment block at the top explaining: (1) bucket must be created manually before `terraform init`, (2) authentication uses `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY` env vars (these are DO Spaces keys, not AWS), (3) change bucket name to match your actual Spaces bucket

**infra/.gitignore:**
- Use the standard Terraform .gitignore from RESEARCH.md (excludes .terraform/, *.tfstate, *.tfvars, crash logs, override files, lock info, CLI config)
- Add a comment noting that `.terraform.lock.hcl` is intentionally NOT ignored (ensures reproducible provider versions)

<task type="auto">
  <name>Task 2: Create variables.tf and terraform.tfvars.example with all configurable values</name>
  <files>infra/variables.tf, infra/terraform.tfvars.example</files>
  <action>
**infra/variables.tf:**

Declare all variables with `description`, `type`, and `default` (where appropriate). Mark `do_token` as `sensitive = true`.

Variables to declare (in this order):
1. `do_token` (string, sensitive, no default) -- DigitalOcean API token
2. `project_name` (string, default "odoo-prod") -- prefix for all resource names
3. `region` (string, default "nyc3") -- DO region for all resources
4. `droplet_size` (string, default "s-2vcpu-4gb") -- droplet size slug
5. `volume_size_gb` (number, default 25) -- Block Storage Volume size in GB
6. `vpc_cidr` (string, default "10.100.0.0/24") -- VPC IP range (safe range outside DO reserved blocks per RESEARCH.md)
7. `use_existing_ssh_key` (bool, default true) -- toggle between existing DO key lookup vs local key upload
8. `ssh_key_name` (string, no default) -- name of existing SSH key in DO account (used when use_existing_ssh_key = true)
9. `ssh_public_key_path` (string, default "~/.ssh/id_ed25519.pub") -- local public key path (used when use_existing_ssh_key = false)
10. `ssh_private_key_path` (string, default "~/.ssh/id_ed25519") -- local private key path for remote-exec provisioner SSH connection
11. `allowed_ssh_ips` (list(string), no default) -- CIDR blocks allowed to SSH into the droplet

**infra/terraform.tfvars.example:**

Create a well-commented template matching RESEARCH.md's example. Include:
- Comment at top: "Copy this file to terraform.tfvars and fill in your values"
- Comment explaining env var alternative for `do_token`: `export DIGITALOCEAN_TOKEN="your-api-token"`
- Comment explaining env vars needed for Spaces backend: `export AWS_ACCESS_KEY_ID="..."` and `export AWS_SECRET_ACCESS_KEY="..."`
- Placeholder values for every variable (use realistic examples, not "CHANGE_ME")
- Comment next to `allowed_ssh_ips` warning to restrict to operator's IP(s)
- Comment noting `ssh_public_key_path` is only used when `use_existing_ssh_key = false`
  </action>
  <verify>
Verify variables and tfvars alignment:
- `grep -c 'variable "' infra/variables.tf` returns 11 (one per variable)
- `grep -q 'sensitive.*true' infra/variables.tf` (do_token is marked sensitive)
- `grep -q 'DIGITALOCEAN_TOKEN' infra/terraform.tfvars.example` (env var documented)
- `grep -q 'AWS_ACCESS_KEY_ID' infra/terraform.tfvars.example` (Spaces keys documented)
- `grep -q 'allowed_ssh_ips' infra/terraform.tfvars.example`
- Every variable name in variables.tf has a corresponding entry in terraform.tfvars.example (except do_token which uses env var)
  </verify>
  <done>
variables.tf declares 11 variables with types, descriptions, defaults (where appropriate), and sensitive flag on do_token. terraform.tfvars.example provides a complete template with realistic placeholder values and comments explaining environment variable alternatives for secrets. Both files are aligned -- every configurable value has both a declaration and a template entry.
  </done>
