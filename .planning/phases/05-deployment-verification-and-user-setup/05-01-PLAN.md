  <action>
Create two bash scripts in odookit/scripts/:

**ssh-tunnel.sh** — SSH tunnel lifecycle manager:
- Accepts `start` or `stop` as first argument (default: start)
- Reads from env vars: INFRA_SSH_HOST (required), INFRA_SSH_PORT (default 9292), INFRA_SSH_USER (default deploy), TUNNEL_LOCAL_PORT (default 8443)
- Uses `set -euo pipefail` and includes `#!/usr/bin/env bash` shebang
- `start`: Kills any existing process on the local port first (lsof -ti:PORT | xargs kill), waits 1s, then opens tunnel with `ssh -f -N -L ${LOCAL_PORT}:localhost:443 -p ${REMOTE_PORT} ${SSH_OPTS} ${USER}@${HOST}`
- SSH options: `-o StrictHostKeyChecking=accept-new -o ServerAliveInterval=60 -o ServerAliveCountMax=3`
- `stop`: Kills process on local port, prints confirmation
- Prints status messages for both start and stop
- Make executable (chmod +x)

**verify-backup.sh** — Backup trigger and verification over SSH:
- Reads same SSH env vars (INFRA_SSH_HOST, INFRA_SSH_PORT, INFRA_SSH_USER)
- Uses `set -euo pipefail`
- Step 1: Trigger a manual backup via `ssh -p PORT USER@HOST "sudo /opt/odoo/scripts/06-backup-daily.sh"`
- Step 2: Read backup-status.json via SSH, check status is 0 (Nagios OK convention)
- Step 3: Trigger offsite sync via `ssh -p PORT USER@HOST "sudo /opt/odoo/scripts/07-sync-offsite.sh"`
- Step 4: Read sync-status.json via SSH, check status is 0
- Step 5: Verify a backup file exists for today's date in /mnt/odoo-prod-data/backups/daily/
- Uses SSH option `-o StrictHostKeyChecking=accept-new`
- Prints PASS/FAIL for each step with a final summary
- Exit code 0 if all pass, 1 if any fail (but continue checking all steps)
- Make executable (chmod +x)
  </action>

  </verify>
  <done>Both scripts pass bash syntax check and are executable. ssh-tunnel.sh supports start/stop with env var configuration. verify-backup.sh triggers backup + sync and checks status files over SSH.</done>
